Design Notes & Key Changes

①
[FIX] h1 heading — Page title changed from <h4> to <h1 class="page-title"> for correct document outline and screen reader compatibility across all pages.
②
[FIX] role="main" — .content-area div gains role="main" (or converted to <main>). Authenticated layout currently has no main landmark.
③
[FIX] Quick action tiles — "Export Excel" and "Import Excel" tiles now open Bootstrap modals directly instead of navigating to /trainings. This matches user intent and avoids confusion.
④
[FIX] aria-current="page" — Active sidebar nav item gets aria-current="page" attribute for screen reader compatibility.
⑤
[NEW] RecentActivity panel — Shows latest user actions (new personnel, training additions, imports). Provides at-a-glance audit trail. Backend: log model or activity log package.
⑥
[NEW] UpcomingTrainings panel — Shows next 3–5 trainings sorted by date. Reuses existing /api/trainings endpoint with ?sort=start_date&direction=asc. No new backend work needed.
⑦
[NEW] Global quick search — Header search (⌘K) for fast navigation to personnel or trainings. Helps power users jump directly to records without sidebar navigation.
⑧
[FIX] aria-expanded on sidebar toggle — Menu button correctly announces open/closed state via aria-expanded and Escape key closes the mobile sidebar.
